Key Design Decisions

Storage

BigQuery — GCP native, handles petabyte scale, SQL-native analytics

Capture

Passive observer — zero changes to existing UCP server or client code

Batching

Async buffer (default 50) — never slows HTTP request/response cycle

Fire-and-forget

Background tasks — analytics failures never block commerce transactions

Transports

REST + MCP + A2A — all 3 UCP communication modes supported

Partial failure

Retries failed rows — minimizes duplication and data loss risk

PII

Optional redaction — configurable per deployment, no PCI-scope data stored
